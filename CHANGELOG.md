## 2.22.2

### 前端

加入了支持ESM的WebWorker安全沙盒
> 未来的插件、提示词、复杂的角色卡，可以像 Node.js 项目一样分多个文件编写，打包成 ZIP 分发。  
> 尽管JS编写者可能用 webpack 或类似工具构建，但沙盒什么都不要安装。  
> 很多人用正则是因为没有别的东西能用，正则又差不多足够"Powerful"，代码和终端用户之间是有鸿沟的。  
> 现在有点不一样了，你可以马上改，马上预览，不需要vite，不需要Node，一切都发生在浏览器里。  
> 你还可以问LLM，让它和你一起改。
- 支持 import with/assert 语法，主要就是让你从文本加载提示词用的
- 支持从 ZIP 包导入模块（基本上因为我写了一个类似ESBuild的Bundler）
- 支持定义宿主模块，提供相当的互操作能力。
- 通过postMessage通信还有超时管理，防止耗尽资源以及病毒
- 尽管做不到可审计（Function可用），但它安全
- 声明式权限管理：网络、存储、WASM
- 10.1KB after minified，但已经比任何其它实现小了

修复模型能力检测代码的bug
修改了工具返回值从done到success (兼容)
修复Edit工具的replaceAll不起作用的问题
修复Grep工具的输入必须是目录/未处理换行/未格式化响应的问题

修复了某些情况日志展示报错的问题
等待工具调用时加入setTimeout和beforeunload的hook防止刷新时数据丢失
修复SSE resume导致UI渲染错误的问题
修复CSS分块导致丢样式的问题

### 通用/工具

## 2.22.0 (6/25)

### 破坏性更改

工具名称全部改为 PascalCase
> 之前版本所有带工具的对话都无法在新版本中继续
> 你仍然可以查看它们

JSON路径由 dot path 改为 JSON Pointer (RFC 6901) + append extension
> 具体看 JSON Editor 工具和 UpdateVariable 工具全新的语法
> 彻底解决了之前模型不用和瞎用的问题
> 设置选项中的 dot path 也需要你手动改成 JSON Pointer
> 它们会自动标红

### 前端

实现PWA，现在你可以把AiChat安装为PWA了
- 安装之前，请把 设置 > 数据 > 远程文件缓存 打开
- assets 是实时缓存，而非预缓存，所以离线使用时异步模块可能无法加载，如 mermaid

单选框使用分段按钮（琴键开关），大概更容易看出这是单选  
全局工具调用批准现在可以指定工具名称，之前只能全部允许  
工具调用块支持自定义标题，例如 ReadFile 会显示文件名  
支持读写JPEG注释。酒馆只支持PNG角色卡，这是为未来角色卡嵌入JPEG预留的API，而不是扩展兼容性

优化标题生成
- 强制JSON
- 优化设置标题工具的提示词
- 支持清除对话标题

优化了日志查看页面的操作
修复了JSON编辑器的性能问题，现在可以流畅的编辑百K级别的JSON，基本满足日常所需

加入了 Grep Append 等全新文件操作工具
- 在虚拟文件系统上进行这些操作，不依赖本机程序

修改了 SetTimeout 的功能
- 它现在更类似QTE，而不是在发送消息时告诉LLM现在几点了
- 当然，你是上帝，所以其实有不少方法能绕过它，我也wontfix

加入了工具、技能、MCP管理抽屉
> 功能越来越多，过段时间我可能又要重构设置窗口了……
> 目前，你需要在工具按钮上右键(PC)或长按(移动端)来打开管理界面

- 管理模型在当前对话中能“激活”和“使用”的工具
- 你可以添加任意MCP服务器，每个服务器将注册为模块，你可以让模型自行激活或在UI中手动管理
- 设置【新对话的默认值】需要通过系统提示的高级语法（在文档中）
- 加入只读文件访问工具组（手动开启）
- 删除了 use_tool, revoke_tool, list_tools 命令
- 记忆和追问工具不再需要靠命令激活
- 删除了 registerDefaultTool API，注册工具现在必须通过 registerTools 函数
- 增加了独立的 Skill 工具用于激活技能，通过 registerSkill 函数

加入了插件系统及UI
- 之前版本插件只能在构建前开关
- 现在可以通过UI开关（刷新页面生效）

### 插件

加入了一些插件，它们都是默认关闭的，你需要手动启用：

【TaskList、结构化任务】
- 显示和操作待办清单，主要是给人看的

【RPG管线Lite】
- /say 命令移动到这里

【JSON编辑工具】
- 基于约束采样，让LLM编辑JSON永远不出现语法错误
- 提供基于JSON Pointer的增量更新工具
- 删除网页版JSON Editor中所有的AI功能
- 支持根据 schema 文件验证 json 文件
- 支持 json5: 可以用单引号，可以裸键名，可以用注释以及各种计数法

【交互式文件上传/下载】
- 手动上传文件给LLM
- 打包下载任意文件系统中的文件
- （终于可以操作OPFS的数据了！）

【外部插件API】
- 终于有插件API了
- 你不需要编译也可以通过 `window.AiChatAPI` 注册自己的工具、渲染器、命令等
- 尽管如此，我不保证任何形式的向后兼容

### 后端
修复WS重连时锁定状态没有更新可能导致数据丢失的问题  
修复分支对话不同步的问题  
新增后台进程工具，可以在后台运行程序，如 npm run dev，不阻塞LLM
