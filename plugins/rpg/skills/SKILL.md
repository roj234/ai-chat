## 故事生成流程

> 使用 `ValidateJson` 工具验证文件是否符合 Schema。  
> 使用 `EditJson` 工具在用户要求修改时进行增量更新。  
> 多和用户交流确定故事梗概，不要急于修改文件；适当使用 `AskUser` 工具。  
> 在编写过程中，使用 `RollDice` 工具生成随机数来引导你的行为。
> - 如果用户说随便/自由发挥，那么**必须**使用该工具
> - 例如，随机生成世界时，通过 RollDice 从多个选项中确定世界类型
> - 例如：生成故事线时，获取一个 0-100 的随机数当作运气值/邪恶值
> 如果没有上述工具，激活 `JsonEditor` 和 `InteractiveSimulation` 模块。
> 
> 不要把生成的文件放在这个文件夹里。
> 
> 按下方步骤进行，完成一步再进入下一步。  
> 在每一步开始之前，读取对应 Schema 文件。 

### 1. 构造世界观

Schema: `world.schema.json5`

针对其中大部分需要详细描述的文本字段（age, geography, style, leveling, social）等，请务必采用以下类 Markdown 排版规范输出长文本：
1. **首段定调**：每一项内容开始前，必须先写一段或一句话进行核心概括。
2. **严格分点**：核心概括后，务必使用 "1. XXX:" 配合其下属的 "- yy" 列表符进行层级拆解。
3. **信息密度**：切忌空洞乏味，必须填充详细的设定细节。

参考排版范例（在对应字段中应用）：  
总结性的一句话概括。
1. 一级核心设定：
    - 二级解释A：详尽说明
    - 二级解释B：详尽说明
2. 另一个核心设定：
   ...

### 2. 构造角色

角色可以是主角（PC），也可以是 NPC

Schema: `character.schema.json5`。

#### 角色设计准则
1. **世界观契合度**：严格使用世界观的力量体系层级、专属名词和社会法则，不要引入新设定。
2. **文风一致性**：描述文风继承世界观的基调，背景故事要有张力。
3. **属性严谨**：严格对照世界观的属性体系（ID、值域、基准），赋予合乎角色背景的逻辑数值。

#### skills 字段

每个技能需要 `cost_and_limit`——写出具有**叙事张力**的限制，而非单纯数字消耗。例如：
- ✅ "只能在雷雨夜使用，否则反噬自身"
- ✅ "每次使用会失去一段珍贵记忆"
- ❌ "消耗 40MP，每天一次"

#### attributes 字段
key 为世界观的属性 ID，value 遵守对应 rank_rule。  
如世界观定义 `HP: 0-100, 常人为75`，则角色 HP 应在此区间内赋予合理值。

### 3. 故事线(主要剧情)

为冒险生成故事线，在冒险过程中随回合数推进逐渐浮出水面。  
故事线可能有多条，或明或暗，亦正亦邪。

Schema: `plot-arc.schema.json5`, `plot-arc-node.schema.json5`。

#### 剧情类型分级
- **主线级危机**：关乎世界格局或区域存亡，潜伏期长。第一个节点 `expected_turn=0`。
- **个人宿命线**：针对主角身世和弱点的阴谋，扎心且致命。
- **变数与风波**：较小规模，突发诡异事件、帮派火拼、失控的古代遗物等。
- 虽然例子都是负面，但也可以有一定正面内容。

#### 随机化
记得用 `RollDice`
```js
function getActToneByRandom() {
	const dice = roll(0, 100);
	if (dice < 15) {
		return "【天降大运 / 史诗奇遇】（大吉）：这是一次极其罕见的正面机遇。主角可能会继承陨落神明的遗产、遇到隐世高人求着收徒、或者无意间解开了某个宝藏的封印。剧情重点在于争夺机缘和探索未知，而非生死危机。";
	} else if (dice < 40) {
		return "【迷雾重重 / 喜剧风波】（中吉/波折）：没有立刻丧命的危险，更多是荒诞、神秘或有趣的展开。例如：一只会说话的猫非要认主角当主子引发的闹剧、突然天降一份离谱的契约、或者是某种无害但极为诡异的环境异变。";
	} else if (dice < 70) {
		return "【宿命纠缠 / 阴谋渐起】（中平/暗流）：经典的 TRPG 开局。看似平静的日常下暗流涌动，主角因为某个微小的举动卷入了大势力的斗争，或者一个针对他弱点的阴谋正在悄悄收网。危机与机遇并存。";
	} else if (dice < 90) {
		return "【凶煞劫难 / 生死一线】（大凶）：具有压迫感的危机开局。深渊入侵、宗门被灭、高额悬赏追杀、或者是自身沾染了某种致命的诅咒。必须步步为营，随时可能丧命。";
	} else {
		return "【天道倾覆 / 绝望死局】（极恶）：世界级的灾难直接砸在主角脸上，或者是最亲近之人的背刺。不需要前期慢慢铺垫，开局就是最高潮的逃亡或绝境求生反杀。";
	}
}
```

#### 动态重规划 （PC 破坏伏笔时） (WIP)
如果 PC 的行为改变了故事线：原定计划作废，反派可能狗急跳墙、将计就计、或引发其它后果。重新规划未到达的节点。

### 4. 开场事件 (Opening Greeting)

Schema: `greeting.schema.json5`

#### 创作原则
1. **切忌平铺直叙**：不要讲述世界观或角色过去！这些 PC 已经知道了。你要做的是直接让动作发生（比如刀锋贴到了脖子上）。
2. **五感拉满**：使用视、听、嗅、触觉来建立场景的真实感。
3. **埋下伏笔**：在描写中自然地融入【故事线】的征兆。
4. **结尾悬念**：最后一段必须抛出一个迫在眉睫的抉择，逼迫 PC 行动。
5. **初始化变量**：以 JSON 形式给出开场时的变量状态。

建议 4000 字左右
