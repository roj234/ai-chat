### 第一步：配置 API （针对小白）

1. 点击侧边栏左下角的扳手图标（移动端需要先点击左上角打开侧边栏），进入设置面板
2. 点击`模型`Tab
3. 填写`API 地址` `API 密钥`、`模型 ID`三项（模型ID在填好前两项之后支持自动补全）
4. 滚动到最底部，查看`应用配置模板`按钮

如果可以点击，那么你需要点击它，然后开始聊天。  
如果你未来修改了API地址或模型ID，也需要重新点击`应用配置模板`（除非你不再是小白）

### 第二步：开始对话

- 在输入框输入消息，按 Enter 发送
- 支持 **Shift+Enter** 换行
- 点击左侧「新建对话」按钮创建新会话

## 斜杠命令

在聊天输入框中输入以下命令（输入 `/help` 查看完整列表）：
命令是支持双引号转义的，这意味着你可以使用 "\n" 来输入一个换行符作为参数
此外，本项目的命令解释器非常宽容，你可以在双引号内输入真正的换行

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有可用命令 |
| `/new` | 创建新对话 |
| `/title <文本>` | 重命名当前对话 |
| `/preset <名称>` | 加载预设配置 |
| `/tools` | 列出可用工具 |
| `/use_tools <工具名>` | 启用指定工具 |
| `/revoke_tools <工具名>` | 禁用指定工具 |

## 界面功能

### 底部输入框工具栏，从左到右

- **预设切换**：快速切换过去保存的预设
- **思考开关**：控制是否启用深度思考
- **工具开关**：控制是否启用工具调用
- **分支按钮**：启用对话分支功能

### Markdown附件引用

输入框支持这三种语法，整行必须只有这些文字：
- `![imageN]` 引用第 N 个通过回形针按钮添加的文件（文本、图片、音频）
- `![blob](blobId)` 引用之前上传到后端的附件，附件长度必须为43个字符
- `![file](文件名)` 引用知识库中的文件

### 消息操作

每条消息支持以下操作（从左到右）：
- **编辑**：编辑消息内容（包括思维链和工具调用）
- **复制**：复制消息文本
- **重新生成**：重新生成当前回复
- **删除**：删除该消息及其后续

### 对话分支

启用分支功能后，你可以：
- 在任何消息处点击“编辑”创建分支，从该点开始新的对话路径
- 在不同分支间自由切换

## 设置面板

设置面板包含很多配置项，大部分都有 title
你可以把鼠标悬浮在上面查看提示

### 数据管理
导出

- 单个纯文本对话以 JSON 格式导出
- 其余对话（多个对话或包含文件）以 ZIP 格式导出
- 如果当前打开了某个对话，那么只导出这一个对话，否则导出全部对话

导入

- 支持导入 JSON 或 ZIP 格式的备份
- 支持导入 SillyTavern（酒馆）角色卡、世界书、预设（JSON/PNG 格式）


### 个性化
- 音效开关（生成结束时播放提示音）
- 在输入框中使用 markdown 图像语法引用上传的附件
- 合并连续的工具调用以节省UI空间（仅渲染，但会导致无法编辑）

### 外观
- 背景图片设置
- 文本字体设置

### 角色
- 在这里管理你的角色卡，世界书，预设，以及你的名字和介绍
- 角色卡，世界书，预设，都有响应式编辑器，它们可能在PC上更好操作，你需要仔细查看 title 来操作他们
- 虽然我说的很简略，但是它们肯定比酒馆上手门槛低，而且好用，只要你当里面那些写着“未实现”的功能不存在

### 提示词
- 如果你玩角色扮演，建议你把系统提示改成`--- 回车 ---`，如果你用的预设里面有所谓的“人造思维链”，那么把`推理预算`改成手动也许有一定意义
- 移除历史思维链，基本上也是RP才需要动，你可以打开`仅手动CoT`

## Markdown 渲染

AiChat 使用自研的流式 Markdown 解析器，支持：
- **正确的加粗**：正确渲染 `**"中文引号加粗"**`（按 CommonMark 规范不应加粗，但 LLM 输出中很常见）
- **KaTeX 数学公式**：`$...$` 行内公式，`$$...$$` 块级公式
- **Mermaid 图表**：流程图、时序图等
- **代码高亮**：基于 highlight.js 的语法高亮
- 大部分标准 Markdown 语法
